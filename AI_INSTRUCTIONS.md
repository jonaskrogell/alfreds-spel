## AI Agent Repository Rules

When evaluating tasks or generating code in this repository as an AI assistant, please always adhere to the following rules:

1. **Keep the README Updated:** Whenever you create a new game or introduce significant modifications to an existing game (such as major mechanical additions or sweeping visual overhauls), proactively update the `README.md` file's "Games Included" section to reflect these changes.

2. **Maintain Screenshots:** If you add a new game or dramatically alter a game's look, capture new screenshots using headless Chrome (e.g., `Google Chrome --headless --screenshot`) and update the corresponding `.png` images referenced in the `README.md`.

3. **Check Accuracy:** Ensure that whenever you summarize or list out titles, emojis, controls, and technical descriptions, they accurately match the real state of code in the target game's local `index.html` file.

4. **Commit Changes:** Once a task or fix is successfully completed and verified, you must stage all changes and commit them to the git repository. Use a clear, concise, and descriptive commit message in English that summarizes the changes.
